# EngramHub

> **The Visual Dashboard for Engram's Semantic Memory.**

EngramHub is a web application (Single Page Application) designed to serve as the graphical interface for the **Engram CLI** system. Its primary goal is to solve knowledge fragmentation across development teams by making the persistent memory generated by AI agents (architectural decisions, resolved bugs, code patterns) visible, governable, and easily searchable.

## 🚀 Key Features

- **Global Timeline:** A social-media style feed displaying agent observations chronologically. Includes context/project filters and cursor-based pagination.
- **Fast Semantic Search:** Find past solutions instantly using the SQLite FTS5 engine built into the Engram backend, complete with syntax-highlighted search snippets.
- **Sync Hub (Team Synchronization):** A visual control panel to package, export, and import knowledge "Chunks" across different team members, facilitating collective learning.
- **Advanced Markdown Support:** Full rendering of agent observations, including GitHub Flavored tables and syntax highlighting for code blocks via `highlight.js`.
- **Aesthetic Design:** A clean, productivity-focused interface based on the popular **Catppuccin Mocha** dark color palette.

## 🏗️ Architecture & Technologies

EngramHub operates on a local client-server model, assuming the user has the **Engram** binary running on their machine.

- **Frontend:** Built with **React 18**, **TypeScript**, and **Vite** for blazing-fast development.
- **Styling:** **Tailwind CSS v3** configured with custom Catppuccin theme colors.
- **Routing:** `react-router-dom` for navigation between Timeline, Search, and Sync views.
- **API Communication (The CORS Proxy):**
  Since the local Engram HTTP server (Go) runs on `http://127.0.0.1:7437` and by default **does not send CORS headers**, this SPA utilizes a **Vite Development Proxy** (`vite.config.ts`). 
  
  All client requests (via Axios) are made to the relative path `/api/engram/*`. Vite intercepts these calls, strips the `/api/engram` prefix, and forwards them cleanly to the Go backend, proactively avoiding any same-origin policy blocks in the browser during development.

## 📋 Prerequisites

To run EngramHub locally, you need:

1.  **Node.js** (v18 or higher) and **npm**.
2.  **Engram CLI:** You must have the Engram binary installed and its local server running on the default port (`7437`). If the engine is not running, the interface will display a connection error banner, though it will remain navigable.

## 🛠️ Installation & Usage (Development)

1.  **Clone the repository** and navigate to the frontend directory:
    ```bash
    git clone <repository-url>
    cd eng-hub-ui
    ```

2.  **Install dependencies:**
    ```bash
    npm install
    ```

3.  **Start the development server:**
    ```bash
    npm run dev
    ```

4.  **Open the application:**
    Vite will provide a local URL. Open your browser to:
    ```text
    http://localhost:5173
    ```
    *(The interface will immediately begin consuming data from your local database via the proxy).*

## ⚠️ Production Notes

The proxy configured in `vite.config.ts` is an **exclusive tool for the development environment** (`npm run dev`).

If you build the application for production (`npm run build`):
- **Isolated Deployment:** If you plan to serve the resulting `dist/` folder from a traditional web server (Nginx, Apache) on a different port than Engram, **you must configure a reverse proxy** to handle request forwarding just as Vite does.
- **Embedded in Go (Zero Dependencies):** If the ultimate plan is to embed the generated static files (`dist/`) directly within the compiled Go binary using the `//go:embed` directive, the frontend and backend will share the same origin (`:7437`). In that scenario, CORS issues naturally disappear, and the proxy is no longer necessary.
